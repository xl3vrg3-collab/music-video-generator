OVERVIEW
--------
This tool takes any song (MP3/WAV) and a style prompt, then generates a full
AI music video using Grok's video generation model. The entire pipeline is
automated: audio analysis, scene planning, video generation, and final
stitching with crossfade transitions and your audio overlaid.

No video editing skills required. Upload a song, describe the vibe, get a
music video.


HOW IT WORKS (PIPELINE)
-----------------------

  Song File (.mp3/.wav)
       |
       v
  [1] AUDIO ANALYSIS (librosa / numpy)
       - Detects BPM and beat timestamps
       - Maps energy curve over time (loud vs quiet sections)
       - Segments the track into sections:
           INTRO    — low energy opening
           VERSE    — moderate energy, storytelling
           CHORUS   — peak energy, climactic
           BRIDGE   — transition, usually different feel
           OUTRO    — wind down, closing
       |
       v
  [2] SCENE PLANNING (scene_planner.py)
       - Divides the song into ~8-second scenes (Grok video clip length)
       - Generates a unique visual prompt for each scene based on:
           * Your style input ("cyberpunk neon city rain")
           * Section type (intro gets wide establishing shots,
             chorus gets high-energy dynamic shots, bridge gets dreamy)
           * Energy level (high energy = fast motion, low = slow/calm)
       - Adds cinematic modifiers: camera angles, lighting, mood
       - Example output for a 3-minute song: ~22 scenes
       |
       v
  [3] VIDEO GENERATION (Grok grok-imagine-video API)
       - Sends each scene prompt to xAI's video generation model
       - Each clip is ~8 seconds at high quality
       - Runs up to 3 generations concurrently for speed
       - Async polling: submit request, poll every 5 seconds for completion
       - If video generation fails for a scene, falls back to:
           * Generate a still image (grok-imagine-image)
           * Apply Ken Burns effect (slow zoom/pan) via ffmpeg
           * Result: smooth animated clip from a single image
       - Downloads all clips to output/clips/
       |
       v
  [4] VIDEO STITCHING (ffmpeg)
       - Takes all generated clips in order
       - Applies 0.5-second crossfade transitions between each clip
       - Overlays the original audio track on the combined video
       - Adds 1-second fade-in at start and 2-second fade-out at end
       - Outputs final music video as MP4 (H.264 video, AAC audio)
       |
       v
  Final Output: music_video.mp4


SYSTEM REQUIREMENTS
-------------------
  - Python 3.10 or newer
  - ffmpeg installed and on your PATH
      Windows:  winget install ffmpeg
      Mac:      brew install ffmpeg
      Linux:    sudo apt install ffmpeg
  - xAI API key with access to grok-imagine-video model
  - Internet connection (for API calls)
  - ~500MB free disk space per music video


INSTALLATION
------------
  1. Clone or download this project

  2. Install Python dependencies:
       pip install -r requirements.txt

     Dependencies:
       - requests     (HTTP client for Grok API)
       - librosa      (audio analysis — optional but recommended)
       - numpy        (numerical processing)

     If librosa fails to install (common on Windows), the tool will
     fall back to basic wave file analysis using Python's built-in
     wave module. Beat detection will be less accurate but functional.

  3. Create your .env file:
       cp .env.example .env

     Edit .env and add your xAI API key:
       XAI_API_KEY=your_key_here

  4. Verify ffmpeg is installed:
       ffmpeg -version


USAGE — COMMAND LINE
--------------------
  Basic:
    python generate.py --song mysong.mp3 --style "cyberpunk neon city"

  Full options:
    python generate.py \
      --song   path/to/song.mp3 \
      --style  "dark moody cinematic rain neon lights" \
      --output path/to/output.mp4 \
      --seed   42 \
      --dry-run

  Options:
    --song      Path to your audio file (MP3 or WAV)
    --style     Visual style description — be descriptive!
    --output    Output file path (default: output/final_video.mp4)
    --seed      Random seed for reproducible scene planning
    --dry-run   Analyze audio and plan scenes WITHOUT generating videos
                (useful to preview the scene prompts before spending API credits)

  Typical run:
    1. First do a dry run to preview the plan:
         python generate.py --song track.mp3 --style "synthwave retro" --dry-run

    2. Review the scene prompts it generates

    3. Run for real:
         python generate.py --song track.mp3 --style "synthwave retro"

    4. Wait for generation (a 3-minute song takes ~10-15 minutes)

    5. Find your video at output/final_video.mp4

STYLE PROMPT TIPS
-----------------
  Your style prompt drives the visual look of every scene. Be descriptive
  but consistent. The tool automatically varies shots per section, so you
  just need to set the overall vibe.

  Good prompts:
    "dark cyberpunk city at night, neon signs, rain, moody lighting"
    "ethereal forest, golden hour, mist, dreamy soft focus"
    "abstract geometric shapes, bold colors, high contrast, minimal"
    "underwater ocean scene, bioluminescent creatures, deep blue"
    "gritty urban streets, graffiti, skateboarding, handheld camera"
    "outer space, nebulas, astronaut floating, cinematic wide shots"

  Tips:
    - Include lighting: "neon", "golden hour", "moody", "harsh shadows"
    - Include mood: "dark", "ethereal", "energetic", "melancholic"
    - Include camera: "cinematic", "handheld", "aerial", "macro"
    - Keep it to 10-20 words — too long and the model loses focus
    - The tool adds section-specific modifiers automatically
      (you don't need to specify "wide shot" for intro, etc.)

SCENE PLANNING DETAILS
----------------------
  The scene planner adapts visuals to the music structure:

  INTRO (low energy):
    - Wide establishing shots
    - Slow camera movement
    - Setting the scene

  VERSE (moderate energy):
    - Medium shots, closer framing
    - Steady camera with gentle movement
    - Storytelling visuals

  CHORUS (high energy):
    - Dynamic shots, fast movement
    - Bold colors, high contrast
    - Energetic, climactic visuals

  BRIDGE (transitional):
    - Abstract or dreamy visuals
    - Different color palette
    - Contemplative mood

  OUTRO (fading energy):
    - Pull back to wide shots
    - Slower movement
    - Fading, closing visuals

  Within each section, energy levels further modify the prompts:
    - High energy: "dynamic movement, vibrant, intense"
    - Medium energy: "steady, flowing, balanced"
    - Low energy: "still, calm, minimal movement"


TROUBLESHOOTING
---------------
  "ffmpeg not found"
    Install ffmpeg and ensure it's on your PATH.

  "librosa not found"
    pip install librosa
    Or: the tool will fall back to basic wave analysis automatically.

  "API key error"
    Check your .env file has a valid XAI_API_KEY.

  "Video generation failed"
    The tool automatically retries and falls back to image + Ken Burns.
    If all scenes fail, check your xAI API quota/billing.

  "Video is silent"
    Ensure your input file is a valid MP3 or WAV with audio data.

  Slow generation:
    Each 8-second video clip takes 30-90 seconds to generate.
    A 3-minute song with ~22 clips takes 10-15 minutes.
    Up to 3 clips generate concurrently.
