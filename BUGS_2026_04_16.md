## BUG 1: [object Object] broken URLs in character card thumbnails
**Severity:** HIGH — affects all character sheets
**Location:** JS client code that constructs thumbnail URLs from sheet generation response
**Symptom:** Server logs show `GET /[object%20Object]?t=...` 404 errors. The character card thumbnail and the "Sheet 1" image next to the character name are broken.
**Root cause:** The response object is being used directly as a URL string instead of extracting the `.url` or `.sheet_url` property.
**Server log evidence:**
```
[server] "GET /[object%20Object]?t=1776347703217 HTTP/1.1" 404 -
```

## BUG 3: Sheet prompt hardcodes "photorealistic" — ignores project visual style
**Severity:** HIGH — breaks style consistency for animated/anime projects
**Location:** `server.py:8493-8513` — `_build_pos_sheet_prompt()` method, `has_ref_photo=True` branch
**Symptom:** All generated sheets come back as 3D figurine/photorealistic regardless of project visual style setting (Animated, Noir, etc.)
**Root cause:** When a reference photo is present, the prompt says "Photorealistic" and "the person in the reference image" — it never reads the project's visual style or the character's physical description. The physical description (which contains "Makoto Shinkai anime realism") is suppressed when a ref photo exists.
**Fix:** Read the project visual style (from autosave/brief data or project_style.json) and inject it into the sheet prompt. Replace "photorealistic" with the actual style. Replace "the person" with "the character/subject" for non-human characters.

## BUG 7: Character sheets show duplicate angles instead of full rotation
**Severity:** MEDIUM — limits character consistency tools
**Location:** `server.py:8525-8528` — `_build_pos_sheet_prompt()`, sheet prompt text
**Symptom:** Gemini generates 6 images but only 3 unique angles (front, 3/4, side) duplicated in two rows. Should be a full 360° turnaround: front, front-3/4, right side, back-3/4, back, left side.
**Fix:** Change prompt from "front view, three-quarter view, side profile" to "six views showing full rotation: front view, front three-quarter view, right side profile, back three-quarter view, back view, left side profile."
**Reference:** User shared https://x.com/azed_ai/status/2044007534961590493 as target format.

## BUG 8: Gemini edit mode locks to reference photo style, ignores prompted style
**Severity:** HIGH — blocks anime/stylized projects
**Location:** `server.py:8396-8406` — Gemini edit path in sheet generator
**Symptom:** Even with "high-end anime style" in the prompt, Gemini edit mode reproduces the reference image's 3D figurine aesthetic. The reference photo dominates the output style.
**Root cause:** `gemini_edit_image` treats the reference as the primary style anchor. It preserves the look of the input, not the text prompt's style direction.
**Fix options:**
  1. For stylized projects: generate text-only (no ref) using a style-focused prompt, then validate identity in a separate pass
  2. Use a different model for anime styles (FLUX with anime LoRA, or Kling's style transfer)
  3. Two-pass: generate the turnaround in target style, then use Gemini edit to inject identity features (moon, eyes, hoodie) from the ref
