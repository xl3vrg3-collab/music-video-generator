01

Overview

LUMN Studio turns one creative prompt into a finished cinematic film. No crew, no budget, no render farm. Write what you see in your head, and the system builds it.

What It Is

A unified AI film studio running locally in your browser. Describe a movie — LUMN extracts every character, costume, environment, and prop, generates canonical reference sheets, composes shot-specific anchor images, renders video clips, and assembles them into a complete film with transitions, audio, and export.

One master prompt drives everything. The system acts as your AI director, production designer, and editor.

Who It's For

  • Independent filmmakers with a vision but no budget
  • Content creators who want cinematic quality
  • Musicians producing visual albums and music videos
  • Brands building narrative-driven campaigns
  • Anyone who sees movies in their head

Unified Flow

One master prompt drives the entire pipeline. The AI extracts characters, environments, costumes, props, style direction, and a scene breakdown from your description. No manual entity creation required — just write your vision.

Gate Checkpoints

Between stages, soft gates ensure quality. Assets need visual sources before stills generate. Scenes need approved assets for visual consistency. Shots need camera assignments before rendering. Each gate is a quick checkpoint, not a blocker.

Non-Linear Access

While the pipeline flows linearly, you can click any stage in the stepper to jump back and adjust. Re-draft characters, regenerate a single shot, swap an environment reference — then continue forward. Changes propagate downstream.

03

Workspace

A stage-based interface where each screen answers one question. Progressive disclosure keeps things simple; Power Mode unlocks everything.

Stage Stepper

A persistent 5-step progress bar across the top of the workspace. Each step shows a numbered dot and label. Completed stages get a green checkmark. The active stage is highlighted. Click any step to jump to it.

Below the stepper, a description line reads: Step 2 of 5 — Lock your cast, wardrobe, and locations

Bottom Action Bar

A fixed bar at the bottom of the viewport with one primary CTA per stage. The CTA label and action change with each stage: "Generate Project Draft" on Brief, "Render Final Clips" on Render. A back button sits alongside for navigation.

Inspector Panel

A 320px sidebar that slides in from the right when you click any asset, scene, or shot card. Shows full detail editor for the selected item. Close with × or Escape. On mobile (<1024px), it becomes a bottom sheet instead.

Toggle with the PWR button in the toolbar. Power Mode does not add features — it removes progressive disclosure for experienced users:

fal.ai Runtime

Serverless GPUQueueStorage

The infrastructure layer. fal.ai hosts Gemini and Kling behind a unified queue, streams progress updates, stores generated frames and clips on a CDN, and gives LUMN a single credential to pay for. One API surface, two models, zero orchestration.

Availability99%

API Contract That Shapes Architecture

The fal.ai surface dictates the shape of LUMN's pipeline:

EndpointBehaviorImplication
gemini_generate_imageText-only, no referencesUsed for the first canonical sheet when nothing exists yet
gemini_edit_imageText + up to N reference imagesThe composer — builds anchors by stacking character + costume + environment refs
kling_image_to_videoOne first-frame image, text promptRequires a Gemini-composed anchor as the starting frame
All callsAsynchronous queue + webhooksPipeline state machine persists across restarts

This is why LUMN draws a hard line between canonical sheets (permanent PromptOS-tracked reference images) and composed anchors (temporary per-shot first frames). A sheet is a source; an anchor is a derivative. The anchor is what Kling sees.

Location, description, architecture, weather, time of day, lighting, atmosphere, key props, architecture notes, material notes. Environment sheets become the canonical wide for every shot set there — same geometry, same light, every time.

Costumes

Upper body, lower body, footwear, accessories, color palette, material, wear level, texture notes. Linked to characters so the same outfit appears identically across scenes. Wardrobe continuity is an entity invariant, not a hope.

Props

Hero objects that appear on-screen. Category + description drive a dedicated prop sheet. Importance levels (hero / supporting / background) determine whether a prop earns its own generation or just lives in the environment sheet.

description, category, importance

Canonical Sheets vs Composed Anchors

This distinction is the architectural foundation of LUMN's visual pipeline:

Canonical SheetsComposed Anchors
Permanent PromptOS-tracked reference imagesTemporary, per-shot composites
Multiple views per asset (face, full body, 3/4, etc.)One image per shot, derived from 2–5 canonical refs
Source of truth — never driftDerived from canonical sheets — always recomposable
Generated via gemini_generate_imageComposed via gemini_edit_image with multi-ref input
Stored in output/prompt_os/sheets/Stored in output/pipeline/anchors/
User approves individually before downstream useRejected anchors are recomposed from the same sheets

If an anchor looks wrong, you never edit the anchor. You go back to the PromptOS record, adjust a field, regenerate the sheet, and recompose. Identity lives in images, not in prompt text — refs carry who and what, prompts carry camera and pose.

Lock Reference

Protects the input source. A locked reference means the uploaded photo or inspired-by image won't be accidentally overwritten. Input-side protection.

Lock Asset

Protects the generated output. A locked asset's canonical sheets survive batch regeneration. Output-side protection. Both locks coexist independently.

Production Pipeline

The V6 unified pipeline. One master prompt triggers a 12-state production state machine that drives extraction, sheet generation, shot planning, anchor composition, and video rendering through the fal.ai engine stack.

Master Prompt Extraction

Claude Sonnet parses your creative prompt into structured PromptOS records. It identifies and describes every entity needed to produce your film:

Characters
Costumes
Environments
Props
Style Bible
Scene Breakdown

If a character is described vaguely ("a man"), the system invents specific visual details that fit the tone. If wardrobe is not mentioned, appropriate wardrobe is inferred. Environments get atmospheric detail. The extraction is always complete enough to generate every frame.

Orange-bordered states are approval checkpoints where the user reviews output before continuing. In auto_advance mode, these gates are skipped for a one-click workflow.

Shot Family Classification

Each shot is classified into a shot family that determines which canonical refs get priority in anchor composition:

wide_establishing — Environment 100%, Character 30%
character_closeup — Character 100%, Costume 20%
emotional_moment — Character 100%, Costume 50%, Environment 30%
wardrobe_reveal — Costume 100%, Character 80%
prop_interaction — Prop 100%, Character 70%
action_scene — Character 80%, Costume 70%, Environment 80%

Anchor Composition

For each shot, the compositor selects 2–5 canonical hero refs based on the shot family priority weights, then generates a composed still image via gemini_edit_image. This anchor becomes the first frame for kling_image_to_video.

If a shot anchor is rejected, it is recomposed from the same canonical sheets — never edited directly. The canonical-to-anchor relationship is always one-directional. Identity lives in the references, not in prompt text.

Over time, it builds a style vector — a statistical profile of your preferred visual language.

Style Profile

Eight axes define your visual taste: Warm/Cool, Clean/Gritty, Polished/Raw, Calm/Aggressive, Minimal/Dense, Intimate/Epic, Cinematic/Documentary, Symmetrical/Handheld. Take a 2-minute quiz or adjust sliders directly. Your profile influences every generation.

Recommendations

When you create a new scene, the Brain suggests shot type, camera, color grade, and duration based on what scored highest in similar contexts. It's like having an assistant editor who remembers everything you've ever liked.

AutoAgent

An automated optimization loop. The AutoAgent uses the Brain's learned preferences to run multiple generation variants, scoring each against your style profile. It finds optimal settings without manual trial-and-error.

09

Audio

Music-aware editing built on the track you upload. Kling 3.0 generates native scene sound — wind, footsteps, ambient atmosphere — so every clip arrives with diegetic audio already in place. Your song drives the cut. LUMN does the rest.

Song Upload

Drop an MP3, WAV, or M4A into the Brief stage. LUMN reads the file, runs BPM analysis, extracts beat positions, and holds the waveform for the rest of the pipeline to consume.

Beat Detection & Sync

Automatic BPM and beat extraction via the audio analyzer. The waveform visualizer shows every beat marker. Scene boundaries snap to musical beats with three cut modes: on-beat, half-time, double-time.

Auto-Ducking

When vocals are detected in the track, LUMN automatically lowers the music underneath so the performance cuts through. No manual mixing, no keyframing, no sidechain.

Native Scene Sound

Every Kling 3.0 clip is generated with generate_audio on. Footsteps on stone, wind through a canyon, room tone in a concrete corridor — each shot arrives with its own diegetic layer, already synced to the visuals. No SFX library, no Foley pass.

Motivated Transitions

LUMN defaults to motivated cuts — action cuts, look cuts, smash cuts, match cuts. Not generic dissolves. Every transition in the project timeline is a deliberate choice at a beat boundary, not a fallback.

On the roadmap

Per-scene narration input, voice cloning, multi-language dubbing, and stem-based per-instrument visual control are all planned. None of them ship in V6 — we’d rather document a shorter truth than a longer wish list.

10

Export

The Output stage turns rendered shots into a finished film and a delivery-ready file. Platform presets, upscaling, watermarks, and GIFs — every path verified end-to-end.

Platform Presets

One-click export for the platforms that matter:

YouTube TikTok Instagram Reels Twitter / X GIF

Each preset applies the correct aspect ratio (16:9, 9:16), resolution, encoding, and duration cap. No manual ffmpeg.

AI Upscaling

Real-ESRGAN upscales sub-1080p sources to full HD or 4K. The pipeline picks it up automatically when a source is below the target resolution — no manual pass.

Watermarks

Custom text or a logo upload, with position and opacity controls. Apply to the final mix before export. One watermark per project, applied uniformly across every clip.

Beat-Synced Assembly

The assembly step stitches the rendered shots together using the beat map from the uploaded song. Each transition is a deliberate, motivated cut you pick at a beat boundary — hard cut, smash, J-cut, L-cut, or match. What comes out is a single file, audio ducked under any detected vocals, watermark applied.

On the roadmap

Full drag-to-reorder timeline editor, YouTube Shorts and Cinema presets, album art generation, Spotify Canvas, and banner exports are planned but not in V6. Documented in ROADMAP_V6.md.
