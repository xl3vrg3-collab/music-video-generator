Overview

LUMN Studio turns one creative prompt into a finished cinematic film. No crew, no budget, no render farm. Write what you see in your head, and the system builds it.

What It Is

A unified AI film studio running locally in your browser. Describe a movie — LUMN extracts every character, costume, environment, and prop, generates canonical reference sheets, composes shot-specific anchor images, renders video clips, and assembles them into a complete film with transitions, audio, and export.

One master prompt drives everything. The system acts as your AI director, production designer, and editor.

Who It's For

  • Independent filmmakers with a vision but no budget
  • Content creators who want cinematic quality
  • Musicians producing visual albums and music videos
  • Brands building narrative-driven campaigns
  • Anyone who sees movies in their head

Unified Flow

One master prompt drives the entire pipeline. The AI extracts characters, environments, costumes, props, style direction, and a scene breakdown from your description. No manual entity creation required — just write your vision.

Gate Checkpoints

Between stages, soft gates ensure quality. Assets need visual sources before stills generate. Scenes need approved assets for visual consistency. Shots need camera assignments before rendering. Each gate is a quick checkpoint, not a blocker.

Non-Linear Access

While the pipeline flows linearly, you can click any stage in the stepper to jump back and adjust. Re-draft characters, regenerate a single shot, swap an environment reference — then continue forward. Changes propagate downstream.

03

Workspace

A stage-based interface where each screen answers one question. Progressive disclosure keeps things simple; Power Mode unlocks everything.

Stage Stepper

A persistent 7-step progress bar across the top of the workspace. Each step shows a numbered dot and label. Completed stages get a green checkmark. The active stage is highlighted. Click any step to jump to it.

Below the stepper, a description line reads: Step 3 of 7 — Review and approve generated stills

Bottom Action Bar

A fixed bar at the bottom of the viewport with one primary CTA per stage. The CTA label and action change with each stage: "Generate Project Draft" on Brief, "Render Final Clips" on Render. A back button sits alongside for navigation.

Inspector Panel

A 320px sidebar that slides in from the right when you click any asset, scene, or shot card. Shows full detail editor for the selected item. Close with × or Escape. On mobile (<1024px), it becomes a bottom sheet instead.

API Constraints That Shape Architecture

Runway's API dictates key design decisions in the pipeline:

EndpointConstraintImplication
text_to_imageMax 3 @Tag referencesOnly way to compose multiple assets into one frame
image_to_videoONE first-frame image, no @TagsRequires a pre-composed anchor image as input
text_to_videoNo image input at allFallback only, no visual consistency
All endpoints1000-char prompt limitPrompt assembly must compress intelligently

This is why LUMN generates canonical sheets (permanent reference images) and then composes anchor images (temporary shot-specific first frames) from them. The anchor is what gets fed to image_to_video.

Asset System

A three-tier model for visual consistency: Concept → Reference → Generation. Canonical sheets are the permanent source of truth. Anchors are temporary delivery artifacts.

Characters

Physical appearance, role, personality, expression. Attach photo references for visual consistency. Characters get canonical face sheets, full-body views, and expression variants.

Name, physical description, wardrobe notes, default expression, role (protagonist/supporting/extra)

Environments

Locations with atmosphere, lighting, and time of day. Environment sheets include wide establishing, medium angle, texture details, and lighting variants.

Name, description, lighting conditions, atmosphere, time of day, architectural style

Costumes

Wardrobe items linked to characters. Front, side, back views with detail crops. Costume sheets enforce wardrobe continuity across scenes.

Props

Physical objects that appear in scenes. Hero props get dedicated sheets. Importance levels (hero/supporting/background) determine generation priority.

Canonical Sheets vs Composed Anchors

This distinction is the architectural foundation of LUMN's visual pipeline:

Canonical SheetsComposed Anchors
Permanent reference imagesTemporary, shot-specific composites
Multiple views per asset (face, full body, 3/4, etc.)One image per shot, derived from 1-3 canonical refs
Source of truth — never driftDerived from canonical sheets — always recomposable
Generated via text_to_image with @TagsComposed via text_to_image selecting priority refs
Stored in output/pipeline/sheets/Stored in output/pipeline/anchors/
User approves individuallyRejected anchors are recomposed from canonical sheets

If an anchor looks wrong, you never edit the anchor. You go back to the canonical sheets, adjust the source, and recompose. The anchor is a derivative, not a source.

Lock Reference

Protects the input source for an asset. A locked reference means the uploaded photo or AI description won't be accidentally changed. This is an input-side protection.

Lock Asset

Protects the generated output. A locked asset's generated stills won't be overwritten by regeneration. This is an output-side protection. Both locks can coexist independently.

The V5 unified pipeline. One master prompt triggers a 12-state production state machine that drives extraction, sheet generation, shot planning, anchor composition, and video rendering.

Shot Family Classification

Each shot is classified into a shot family that determines which canonical refs get priority in anchor composition:

wide_establishing — Environment 100%, Character 30%
character_closeup — Character 100%, Costume 20%
emotional_moment — Character 100%, Costume 50%, Environment 30%
wardrobe_reveal — Costume 100%, Character 80%
prop_interaction — Prop 100%, Character 70%
action_scene — Character 80%, Costume 70%, Environment 80%

Anchor Composition

For each shot, the compositor selects 1-3 canonical hero refs based on the shot family priority weights, then generates a composed still image via text_to_image. This anchor becomes the first frame for image_to_video.

If a shot anchor is rejected, it is recomposed from the same canonical sheets — never edited directly. The canonical-to-anchor relationship is always one-directional.

Over time, it builds a style vector — a statistical profile of your preferred visual language.

Style Profile

Eight axes define your visual taste: Warm/Cool, Clean/Gritty, Polished/Raw, Calm/Aggressive, Minimal/Dense, Intimate/Epic, Cinematic/Documentary, Symmetrical/Handheld. Take a 2-minute quiz or adjust sliders directly. Your profile influences every generation.

Recommendations

When you create a new scene, the Brain suggests shot type, camera, color grade, and duration based on what scored highest in similar contexts. It's like having an assistant editor who remembers everything you've ever liked.

AutoAgent

An automated optimization loop. The AutoAgent uses the Brain's learned preferences to run multiple generation variants, scoring each against your style profile. It finds optimal settings without manual trial-and-error.

09

Audio System

Music-aware editing, AI voice generation, multi-language dubbing, stem separation, and beat-synced cuts. Audio lives in two places: song upload in Brief, mixing in Output.

Track Management

Three track types: Music, Voice/Dialogue, SFX/Ambience. Upload files (MP3, WAV, M4A) or generate with Runway's audio API. Mix levels independently with per-track volume and ducking controls.

Beat Detection & Sync

Automatic BPM analysis and beat extraction. The waveform visualizer shows beat markers. Snap scene boundaries to musical beats with multiple cut modes: on-beat, half-time, double-time.

AI Voice (TTS)

Generate narration and dialogue using 40+ Runway voice presets. Type your script, select a voice, generate. Speech-to-speech transfers voice style from one recording to another.

Voice Dubbing

Upload audio in one language, translate and re-voice to another while preserving voice character. Supports 15 languages including Spanish, French, Japanese, Arabic, and Korean.

Stem Separation

Split a mixed track into drums, bass, vocals, and melody. Use stems for per-instrument visual control — map drum hits to cuts, bass to camera sway, vocals to close-ups.

Transitions

10+ transition types: Dissolve, Crossfade, Hard Cut, Fade Black, Fade White, Wipe, Slide, Glitch, Circular Reveal, Squeeze. Durations auto-scale with clip length — short clips get snappy cuts, long clips get smooth blends.

10

Export

The Output stage has three internal tabs: Assembly (timeline editing), Audio (mixing), and Export (format and delivery). From timeline to distribution in one view.

Assembly Tab

The sequence editor. A canvas timeline with drag-to-reorder clips, drag-edge trimming, scroll-to-pan, zoom control. Storyboard view for visual overview. Playback controller with scrubbing. Text overlays for titles, subtitles, lower thirds.

Audio Tab

Full audio mixing environment. Upload music, generate TTS, separate stems, clone voices, dub to other languages. The scene ruler shows clip boundaries aligned with audio tracks.

Export Tab

Format, quality, and platform settings. One-click presets for every major platform:

YouTube YT Shorts TikTok IG Reel IG Post Twitter/X Cinema GIF

Each preset applies the correct aspect ratio (16:9, 9:16, 1:1), resolution (480p to 4K), encoding, and duration limits. AI upscaling via Real-ESRGAN is available for sub-1080p sources.

Album Art & Branding

Generate album covers (1:1), Spotify Canvas visuals (9:16), and banners (16:9) from your project style. Add watermarks with custom text, logo upload, position, and opacity controls.
